# Pack Optimizer

A design-first HTTP API written in Go that calculates the optimal combination of packs needed to fulfil an order. Pack sizes are configurable at runtime via the API — no restarts or code changes required.

**Live demo:** https://pack-optimizer-go-3f0f2ae79338.herokuapp.com

---

## Tech Stack

- **Go 1.26** with [Goa v3](https://goa.design/) (design-first, code-generated transport layer)
- **Docker** — multi-stage build, image published to Docker Hub (`vukovic96/pack-optimizer`)
- **GitHub Actions** — CI/CD pipeline with build, test, coverage gate, and Docker push
- **Heroku** — live deployment

## How the Algorithm Works

The algorithm is in `internal/calculator/calculator.go` and runs in **two passes**.

**Pass 1** works greedily over the sorted pack sizes:

1. `optimal()` — looks for a pack size where `totalItems / size == 1` (an exact or near-exact single-pack fit).
2. If none found, `closest()` — picks the pack size with the smallest absolute distance to the remaining item count.
3. `findOptimalPack()` — given the chosen size, walks down from the maximum possible pack count, recursively checking if the remainder can be covered by the remaining sizes. This avoids overshoot.
4. Repeats until the item count reaches zero.

**Pass 2** re-runs the same logic on the total items produced by pass 1. This corrects cases where the greedy first pass picked a combination that could be improved — for example, where rounding up to a larger pack in pass 1 creates a tighter solution in pass 2.

Pack sizes are read from `PackSvc.GetSizes()` at call time, so runtime updates via `PUT /packs/sizes` are reflected immediately without any restart.

## Getting Started

### Prerequisites

- Go 1.22+
- Make
- Docker (optional)

### Run locally

```bash
git clone https://github.com/aleksandarv/pack-optimizer.git
cd pack-optimizer

go mod download
make build
./bin/pack-optimizer
```

Then open `http://localhost:8080` in your browser.

**Flags:**

```bash
./bin/pack-optimizer --port=9090 --debug=true
```

### Run with Docker

```bash
docker build -t pack-optimizer .
docker run -p 8080:8080 pack-optimizer
```

### Environment Variables

| Variable | Default | Description |
|----------|---------|-------------|
| `PORT`   | `8080`  | HTTP listen port |
| `DEBUG`  | `false` | Enables verbose debug logging |

Both flags and environment variables are supported — env vars are mapped to flags automatically in `loadFlagsFromEnv()`.

## Development

### Regenerate Goa code

If you modify `design/optimizer.go`, regenerate the transport layer with:

```bash
make generate
```

This runs `goa gen` and overwrites everything under `gen/`. Never edit files in `gen/` by hand.

### Makefile targets

| Target         | Description |
|----------------|-------------|
| `make all`     | fmt + vet + test + coverage + build |
| `make build`   | Compile binary to `./bin/pack-optimizer` |
| `make test`    | Run unit tests (race detector enabled, 30s timeout) |
| `make coverage`| Run tests with HTML coverage report; fails if coverage < 70% |
| `make generate`| Regenerate Goa transport code from design |
| `make docker-build` | Build Docker image |
| `make push`    | Build and push image to Docker Hub |
| `make clean`   | Remove binary and coverage artifacts |

## Testing

Tests live alongside the code they test. The CI pipeline enforces **≥ 70% coverage** — the build fails otherwise.

```bash
make test       # run all tests
make coverage   # run tests + open coverage.html
```

| Package | Coverage focus |
|---------|---------------|
| `internal/calculator` | Correctness of the optimisation algorithm across a wide range of inputs, including large numbers and non-standard pack sizes |
| `internal/pack` | Validation, sorting, defensive copying, concurrent access |
| `internal/logger` | Context injection, request middleware, panic behaviour without logger |

## CI/CD

`.github/workflows/project-build.yaml` runs on every push and PR to `main`:

1. Set up Go 1.26.2
2. `make all` — fmt, vet, test, coverage gate (≥ 70%), build
3. Log in to Docker Hub (via `DOCKER_USERNAME` / `DOCKER_PASSWORD` secrets)
4. `make push` — build and push `vukovic96/pack-optimizer:latest`
